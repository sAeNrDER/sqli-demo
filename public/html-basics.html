HTML Basics

HTML (HyperText Markup Language) is the skeleton of every web page. Understanding how it works is essential for understanding how web attacks — like SQL injection — are possible.

1. Page Structure

Every HTML page follows the same basic skeleton. Tags wrap content and tell the browser how to render it.

<!-- Every page starts with this -->
<!DOCTYPE html>
<html lang="en">

  <head>                        <!-- Metadata (not visible) -->
    <meta charset="UTF-8">
    <title>My Page</title>
    <link rel="stylesheet" href="style.css">
  </head>

  <body>                        <!-- Visible page content -->
    <h1>Hello World</h1>
    <p>This is a paragraph.</p>
  </body>

</html>

2. Common Tags

Tag Purpose Example
<h1>–<h6> Headings (largest to smallest) <h1>Title</h1>
<p> Paragraph of text <p>Text here</p>
<a> Hyperlink <a href="/page">Click</a>
<form> User input container — sends data to server <form action="/login" method="POST">
<input> Text field, password box, checkbox etc. <input type="text" name="user">
<button> Clickable button <button type="submit">Login</button>

3. HTML Forms — How User Data Reaches the Server

Forms are the primary way user input (like usernames and passwords) gets sent to a web server. The action attribute sets the URL, and method sets whether it's a GET or POST request.

<form
  action="/login"      <!-- Where to send the data -->
  method="POST"      <!-- POST = data in body, GET = data in URL -->
>
  <label>Username:</label>
  <input type="text" name="username">

  <label>Password:</label>
  <input type="password" name="password">

  <button type="submit">Log In</button>
</form>
Key insight: When you click "Submit", the browser packages the input field values into an HTTP request and sends them to the server. This is exactly what Burp Suite intercepts — and what SQL injection exploits.

4. GET vs POST

GET Request
GET /search?q=laptops&sort=price HTTP/1.1
Host: shop.example.com

# Data visible in URL
# Logged in browser history
# Good for: searches, filters
POST Request
POST /login HTTP/1.1
Host: shop.example.com
Content-Type: application/x-www-form-urlencoded

username=admin&password=secret
# Data in request body
# Not in URL/history
# Good for: login, forms
